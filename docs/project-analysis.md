# Harness Engineering 项目深度分析报告

## 一、项目概述

Harness Engineering 是一个用于 AI Agent（特别是 Coding Agent）的"驾驭系统工程"框架。它通过约束、反馈回路、工作流控制、工具接口与持续改进，让 Agent **稳定、可靠、可控**地完成复杂任务。

**核心理念**: `Agent = Model + Harness`

**设计哲学**: Human Steer, Agent Execute（人类掌舵，智能体执行）

---

## 二、项目能力分析

### 2.1 核心能力

| 能力 | 说明 | 擅长程度 |
|------|------|----------|
| **任务生命周期管理** | 创建、执行、取消、重试任务 | ⭐⭐⭐⭐⭐ |
| **多 AI Agent 调度** | 统一管理 Claude Code、Hermes、Codex CLI | ⭐⭐⭐⭐⭐ |
| **约束与安全控制** | 安全、质量、架构三维约束 | ⭐⭐⭐⭐⭐ |
| **反馈循环与自动修复** | 验证器、修复器、指标记录 | ⭐⭐⭐⭐ |
| **知识库与模式学习** | 全文搜索、模式匹配、经验积累 | ⭐⭐⭐⭐ |
| **多 Agent 协作** | Agent 注册、消息总线、任务编排 | ⭐⭐⭐⭐ |

### 2.2 擅长场景

#### ✅ 最擅长

1. **带约束的代码生成**
   - 安全约束：扫描硬编码密钥、危险 SQL、不安全导入
   - 质量约束：文件长度限制、测试覆盖检查
   - 架构约束：循环导入检测、分层架构验证

2. **多 AI Agent 统一调度**
   - 一套代码管理多个 AI 工具
   - 适配器模式，易于扩展
   - 统一的任务接口

3. **Go 语言学习与面试**
   - 内置 Go 面试题库
   - 覆盖 GMP、Channel、错误处理等核心知识点
   - 学习 + 实践一体化

4. **企业级 AI 工作流**
   - 弹性系统：熔断器、重试、限流
   - 性能监控：内存、GC、Goroutine 统计
   - 分布式支持：集群管理、Leader 选举

#### ⚠️ 局限性

1. **实时性要求极高** - SQLite 不适合高并发写入
2. **大规模分布式** - 当前为单机架构
3. **非 Go 语言项目** - 部分约束检查依赖 Go 工具链

| **PatternEngine** | 模式学习 | 模式匹配、成功率统计 |
| **Monitor** | 系统监控 | 任务指标、健康检查 |

## 四、功能模块详解

### 4.1 核心功能

#### 任务管理
- 创建、执行、取消、重试任务
- 任务优先级队列
- 并发任务控制（信号量机制）
- 任务状态跟踪和历史记录

#### 多 Agent 适配器
- **Claude Code**: 代码实现、审查、测试
- **Hermes Agent**: HTTP API 调用、会话管理
- **Codex CLI**: 命令行任务执行

#### 约束系统
- 安全约束：扫描硬编码密钥、危险 SQL、不安全导入
- 质量约束：文件长度限制、测试覆盖检查
- 架构约束：循环导入检测、分层架构验证

#### 反馈循环
- 验证器检查任务结果
- 修复器自动修正问题
- 指标记录和分析

### 4.2 高级功能

#### 弹性系统 (pkg/resilience/)
- **熔断器**: 关闭、打开、半开三种状态
- **重试机制**: 指数退避、抖动、可配置条件
- **速率限制器**: 令牌桶和滑动窗口算法
- **弹性管理器**: 统一管理熔断器和限流器

#### 性能监控 (pkg/performance/)
- **性能分析器**: 自动采集内存、GC、Goroutine 指标
- **快照机制**: 定时采集系统状态
- **统计分析**: 平均值、最大值、趋势分析

#### 事件系统 (pkg/event/)
- **事件总线**: 异步事件处理
- **事件存储**: 事件持久化
- **事件处理器**: 事件订阅和处理

#### 任务调度 (pkg/scheduler/)
- **调度器**: 定时任务执行
- **调度类型**: 一次性、间隔、Cron
- **任务管理**: 启用、禁用、立即执行

#### 插件系统 (pkg/plugin/)
- **插件管理器**: 插件加载、卸载
- **插件接口**: 统一的插件规范
- **插件生命周期**: 初始化、启动、停止

#### 分布式支持 (pkg/distributed/)
- **集群管理**: 节点发现、状态同步
- **任务分配**: Round Robin、负载均衡
- **Leader 选举**: 自动故障转移

### 4.3 工具包

#### 模板系统 (pkg/template/)
- 内置模板：Go 实现、审查、测试、重构、面试
- 自定义模板：JSON 格式，易于扩展
- 模板渲染：变量替换、条件判断

#### 脚手架生成器 (pkg/scaffold/)
- 项目结构生成
- 配置文件生成
- 特性支持：数据库、认证、WebSocket、gRPC

#### 批量管理 (pkg/batch/)
- 任务导入导出：JSON、CSV、YAML
- 任务验证：必填字段检查
- 任务合并：替换、追加、更新策略
- 任务过滤排序

#### Web UI (internal/webui/)
- 管理界面：任务列表、适配器状态、统计信息
- API 接口：RESTful 设计
- 实时更新：WebSocket 支持
