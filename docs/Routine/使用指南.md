# Routine 使用指南

## 一、快速开始

### 1. 准备配置文件

创建 `interview.yaml`:

```yaml
name: go-interview
description: Go 后端工程师面试
type: interview

input:
  candidate:
    experience: "3-5 years"
    target_role: "Senior Go Backend Engineer"
  resume: |
    我做过一个高并发API网关，使用 Go + Redis + Etcd + Kafka。
  focus:
    - GMP scheduler
    - goroutine
    - concurrency

agents:
  interviewer:
    role: interviewer
    prompt: |
      你是一位资深Go面试官。
      规则：
      - 每次只问一个问题
      - 根据回答追问
      - 回答正确提升难度
      
  evaluator:
    role: evaluator
    prompt: |
      评估候选人回答：
      输出：correctness/depth/clarity/practical 各1-10分

workflow:
  - step: start
    agent: interviewer
    action: ask_question
  - step: evaluate
    agent: evaluator
    action: evaluate_answer
  - step: loop
    agent: interviewer
    action: ask_question
    until: max_rounds_reached
  - step: report
    agent: knowledge_gap_analyzer
    action: final_review

settings:
  max_rounds: 10
  timeout: 60m
  enable_scoring: true
```

### 2. 运行面试

```go
package main

import (
    "context"
    "fmt"
    "github.com/harness-engineering/harness/pkg/routine"
)

func main() {
    ctx := context.Background()
    
    // 创建管理器
    manager := routine.NewRoutineManager(routine.RoutineManagerConfig{
        StoreType: "memory",
        ConfigDir: "./docs/Routine",
    })
    
    // 加载配置并创建实例
    instance, err := manager.CreateFromConfig(ctx, "interview.yaml", map[string]any{
        "name": "Go面试-张三",
    })
    if err != nil {
        panic(err)
    }
    
    // 启动面试
    manager.Start(ctx, instance.ID)
    
    // 获取第一个问题
    question, _ := manager.GetQuestion(ctx, instance.ID)
    fmt.Println(question)
    
    // 模拟回答
    manager.SubmitAnswer(ctx, instance.ID, 
        "GMP是Go的调度模型，G是goroutine，M是系统线程，P是逻辑处理器...")
    
    // 获取下一个问题
    question, _ = manager.GetQuestion(ctx, instance.ID)
    fmt.Println(question)
    
    // 获取报告
    report, _ := manager.GetReport(ctx, instance.ID)
    fmt.Println(report)
}
```

## 二、三种使用方式

### 方式 1：Go 代码集成

```go
// 1. 创建引擎
engine := routine.NewRoutineEngine(routine.EngineConfig{
    EnableScoring:  true,
    EnableFollowup: true,
})

// 2. 注册 LLM Agent（可选）
provider := routine.NewClaudeProvider("your-api-key", "")
factory := routine.NewLLMAgentFactory(provider)
agents := factory.CreateAll(config)
for name, agent := range agents {
    engine.RegisterAgent(name, agent)
}

// 3. 创建实例
instance, _ := engine.Create(ctx, config)

// 4. 启动
engine.Start(ctx, instance.ID)

// 5. 交互循环
for instance.Status == routine.StatusRunning {
    question, _ := engine.GetNextQuestion(ctx, instance.ID)
    fmt.Println(question)
    
    // 读取用户输入
    var answer string
    fmt.Scanln(&answer)
    
    // 提交回答
    engine.SubmitAnswer(ctx, instance.ID, answer)
}

// 6. 获取报告
report, _ := engine.GetReport(ctx, instance.ID)
```

### 方式 2：CLI 命令

```bash
# 列出可用配置
harness routine list

# 创建面试
harness routine create --config interview.yaml --name "Go面试"

# 启动面试
harness routine start --id routine-xxx

# 提交回答
harness routine answer --id routine-xxx --answer "我的回答..."

# 获取报告
harness routine report --id routine-xxx --format markdown

# 导出报告
harness routine export --id routine-xxx --output report.html
```

### 方式 3：HTTP API

```bash
# 创建面试
curl -X POST http://localhost:8080/api/v1/routines \
  -H "Content-Type: application/json" \
  -d '{"config": "interview.yaml", "name": "Go面试"}'

# 获取问题
curl http://localhost:8080/api/v1/routines/{id}/question

# 提交回答
curl -X POST http://localhost:8080/api/v1/routines/{id}/answer \
  -H "Content-Type: application/json" \
  -d '{"answer": "我的回答..."}'

# 获取报告
curl http://localhost:8080/api/v1/routines/{id}/report?format=html
```

## 三、配置详解

### 完整配置示例

```yaml
# 面试配置
name: advanced-go-interview
description: 高级Go工程师面试
type: interview

# 输入参数
input:
  candidate:
    name: "张三"
    experience: "5 years"
    target_role: "Senior Go Backend Engineer"
  resume: |
    我做过：
    1. 高并发API网关 (Go + Redis + Etcd + Kafka)
    2. 微服务框架 (gRPC + 服务发现)
    3. 分布式缓存系统
  focus:
    - GMP scheduler
    - GC (三色标记、写屏障、STW)
    - concurrency (channel、select、sync)
    - memory management
    - distributed systems

# Agent 定义
agents:
  interviewer:
    role: interviewer
    prompt: |
      你是一位一线大厂资深Go面试官。
      
      面试风格：
      - 专业、直接
      - 不要过度鼓励
      - 更像真实面试反馈
      
      面试规则：
      1. 每次只问一个问题
      2. 必须根据回答追问
      3. 回答模糊 → 追问细节
      4. 回答错误 → 指出问题
      5. 回答很好 → 提升难度
      
      面试范围：
      - Go runtime
      - scheduler (GMP)
      - GC (三色标记、写屏障、STW)
      - concurrency (channel、select、sync)
      - memory management
      
      项目深挖（必问）：
      - 如何做限流？
      - goroutine 如何控制数量？
      - Redis 解决什么问题？
      - Kafka 是否有堆积？
      - 流量突增 10 倍怎么办？
      
      输出格式：
      【面试官】
      问题内容
      （等待用户回答）

  evaluator:
    role: evaluator
    prompt: |
      你是技术评估官。
      
      评估维度：
      - correctness (1-10): 答案正确性
      - depth (1-10): 理解深度
      - clarity (1-10): 表达清晰度
      - practical (1-10): 实践经验
      
      输出格式：
      【评分】
      正确性: X/10
      深度: X/10
      清晰度: X/10
      实用性: X/10
      综合分: XX/100
      
      优点: ...
      不足: ...
      遗漏: ...

  followup_generator:
    role: followup_generator
    prompt: |
      你负责生成追问。
      
      规则：
      - 必须追问薄弱点
      - 优先追问 runtime / GC / scheduler
      - 追问要更深入

  knowledge_gap_analyzer:
    role: knowledge_gap_analyzer
    prompt: |
      面试结束后分析知识盲区。
      
      输出：
      - 技术评级: junior/mid/senior
      - 是否通过: true/false
      - 优势领域: ...
      - 薄弱环节: ...
      - 学习计划:
        - 主题
        - 原因
        - 资源

# 工作流
workflow:
  - step: start_interview
    agent: interviewer
    action: ask_question
  - step: evaluate_answer
    agent: evaluator
    action: evaluate_answer
  - step: generate_followup
    agent: followup_generator
    action: generate_followup
  - step: repeat_loop
    agent: interviewer
    action: ask_question
    until: max_rounds_reached
    max_iter: 10
  - step: final_review
    agent: knowledge_gap_analyzer
    action: final_review

# 设置
settings:
  max_rounds: 10
  timeout: 60m
  auto_evaluate: true
  enable_followup: true
  enable_scoring: true
  output_format: markdown
```

## 四、Agent 角色说明

### 1. Interviewer（面试官）

**职责**：
- 提出技术问题
- 根据回答追问
- 控制面试节奏

**Prompt 要点**：
- 明确面试范围
- 定义追问规则
- 设置输出格式

### 2. Evaluator（评估官）

**职责**：
- 评估回答质量
- 给出评分
- 指出优缺点

**Prompt 要点**：
- 定义评分维度
- 明确评分标准
- 要求具体反馈

### 3. Followup Generator（追问生成器）

**职责**：
- 分析薄弱点
- 生成追问问题
- 深挖知识盲区

**Prompt 要点**：
- 优先追问薄弱点
- 追问要更深入
- 结合评分结果

### 4. Knowledge Gap Analyzer（知识盲区分析器）

**职责**：
- 分析整体表现
- 识别知识盲区
- 生成学习计划

**Prompt 要点**：
- 综合评估
- 明确评级标准
- 提供学习建议

## 五、评分系统

### 四维评分

| 维度 | 说明 | 评分标准 |
|------|------|----------|
| Correctness | 正确性 | 答案是否正确 |
| Depth | 深度 | 是否理解底层原理 |
| Clarity | 清晰度 | 表达是否清楚 |
| Practical | 实用性 | 是否有实际经验 |

### 评级标准

| 评级 | 综合分 | 说明 |
|------|--------|------|
| Junior | < 50 | 基础薄弱 |
| Mid | 50-70 | 基础扎实 |
| Senior | > 70 | 理解深入 |

### 通过标准

- 综合分 >= 60: 通过
- 综合分 < 60: 未通过

## 七、高级用法

### 1. 自定义 Agent

```go
// 实现 AgentExecutor 接口
type MyAgent struct{}

func (a *MyAgent) Name() string { return "my_agent" }
func (a *MyAgent) Role() routine.AgentRole { return "custom" }
func (a *MyAgent) Execute(ctx context.Context, input routine.AgentInput) (*routine.AgentOutput, error) {
    // 自定义逻辑
    return &routine.AgentOutput{
        Content:    "自定义输出",
        NextAction: "next_step",
    }, nil
}

// 注册
engine.RegisterAgent("my_agent", &MyAgent{})
```

### 2. 使用 LLM

```go
// 创建 Claude 提供者
provider := routine.NewClaudeProvider("your-api-key", "claude-sonnet-4-20250514")

// 创建工厂
factory := routine.NewLLMAgentFactory(provider)

// 创建所有 Agent
agents := factory.CreateAll(config)

// 注册到引擎
for name, agent := range agents {
    engine.RegisterAgent(name, agent)
}
```

### 3. 持久化存储

```go
// 文件存储
manager := routine.NewRoutineManager(routine.RoutineManagerConfig{
    StoreType: "file",
    StoreDir:  "./data/routines",
    ConfigDir: "./configs",
})
```

### 4. 批量处理

```go
// 批量创建面试
candidates := []string{"张三", "李四", "王五"}
for _, name := range candidates {
    instance, _ := manager.CreateFromConfig(ctx, "interview.yaml", map[string]any{
        "name": name,
    })
    manager.Start(ctx, instance.ID)
}
```

## 八、常见问题

### Q1: 如何防止 Agent 自问自答？

在 Prompt 中加入：
```
（等待用户回答，绝对不能自问自答）
```

### Q2: 如何控制面试难度？

在 input 中设置 level：
```yaml
input:
  level: "mid"  # junior / mid / senior
```

### Q3: 如何自定义评分标准？

修改 evaluator 的 Prompt：
```yaml
evaluator:
  prompt: |
    评分标准：
    - 90-100: 优秀
    - 70-89: 良好
    - 50-69: 一般
    - <50: 不合格
```

### Q4: 如何导出报告？

```bash
# 文本格式
harness routine report --id xxx --format text

# Markdown 格式
harness routine report --id xxx --format markdown > report.md

# HTML 格式
harness routine report --id xxx --format html --output report.html
```

### Q5: 如何查看历史记录？

```go
instance, _ := manager.GetInstance(ctx, id)
for _, msg := range instance.GetHistory() {
    fmt.Printf("[%s] %s\n", msg.Role, msg.Content)
}
```

## 九、最佳实践

### 1. Prompt 设计

- ✅ 明确角色背景
- ✅ 列出具体规则
- ✅ 定义输出格式
- ❌ 模糊的描述

### 2. 工作流设计

- ✅ 每步职责单一
- ✅ 有明确终止条件
- ✅ 有错误处理
- ❌ 过于复杂的流程

### 3. 评分设计

- ✅ 多维度评分
- ✅ 具体的反馈
- ✅ 可执行的建议
- ❌ 单一分数

### 4. 迭代优化

- ✅ 根据测试结果调整 Prompt
- ✅ 收集反馈改进流程
- ✅ 定期更新题库
- ❌ 一成不变的模板
