## 扣分分析

- **基础题**：回答准确但偏简洁，应主动展开代码示例
- **设计题**：思路正确但缺少边界情况讨论（超时处理、goroutine 泄漏）

## 面试技巧提炼

### 1. 回答结构化
```
结论 → 原理 → 代码示例 → 边界情况 → 项目中的实际应用
```

### 2. 主动展示深度
- 不要只答"是什么"，要答"为什么"和"怎么做"
- 用自己项目的代码举例，比泛泛而谈更有说服力

### 3. 常见追问方向
- "如果并发量很大怎么办？" → 连接池、限流、缓存
- "如果执行失败怎么办？" → 重试、降级、熔断
- "如果内存不够怎么办？" → 流式处理、分页、GC 调优
