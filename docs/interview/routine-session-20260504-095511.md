## Go 语言 `slice` 底层数据结构与 `append` 扩容机制

### 一、底层数据结构

Go 语言中 `slice` 的底层数据结构定义在 `runtime/slice.go` 中，是一个包含三个字段的结构体：

```go
type slice struct {
    array unsafe.Pointer // 指向底层数组的指针
    len   int            // 当前切片的长度（已使用的元素个数）
    cap   int            // 当前切片的容量（底层数组可容纳的最大元素个数）
}
```

**通俗理解**：`slice` 本身不存储数据，它只是一个"视图"或"窗口"，通过 `array` 指针引用一个真正的底层数组。`len` 是这个窗口当前能看到多少元素，`cap` 是这个窗口最大能扩展到多少元素（无需重新分配内存）。

```go
// 演示 slice 底层结构
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:4] // s 的底层 array 指向 arr 索引 1 的位置
// s.len = 3 (元素: [2, 3, 4])
// s.cap = 4 (从索引 1 到 arr 末尾: [2, 3, 4, 5])
```

内存布局示意：
```
arr:  ┌───┬───┬───┬───┬───┐
      │ 1 │ 2 │ 3 │ 4 │ 5 │
      └───┴───┴───┴───┴───┘
           ↑           ↑
           │           │
s.array ───┘    s.len=3  s.cap=4
                (窗口)     (最大窗口)
```

### 二、`append` 触发扩容的条件

**当 `len == cap` 时触发扩容**。即当前切片已满，没有多余的容量来容纳新元素，此时 `append` 必须分配一块更大的底层数组，将原有元素拷贝过去，然后追加新元素。

```go
s := make([]int, 0, 2) // len=0, cap=2
s = append(s, 1)       // len=1, cap=2, 未扩容
s = append(s, 2)       // len=2, cap=2, 未扩容
s = append(s, 3)       // len 将变为 3 > cap(2), 触发扩容!
```

### 三、扩容策略（Go 1.18+）

扩容策略经历了演进，Go 1.18 之后采用更平滑的增长：

```go
// 简化版扩容逻辑（runtime/slice.go 中的 growslice）
// newcap 计算规则：
// 1. 如果所需新容量 > 旧容量 * 2  →  newcap = 所需新容量
// 2. 如果旧容量 < 256              →  newcap = 旧容量 * 2
// 3. 如果旧容量 >= 256             →  平滑增长，公式为：
//    newcap += (newcap + 3*256) / 4  （约 1.25x 增长，逐步趋近 2x）
```

```go
// 演示扩容行为
func growDemo() {
    var s []int

    // 小容量阶段，每次翻倍
    for i := 0; i < 1024; i++ {
        beforeCap := cap(s)
        s = append(s, i)
        if cap(s) != beforeCap {
            fmt.Printf("len=%d, oldcap=%d, newcap=%d, ratio=%.2f\n",
                len(s), beforeCap, cap(s), float64(cap(s))/float64(beforeCap))
        }
    }
}
// 输出示例（运行结果因内存对齐可能略有差异）：
// len=1,   oldcap=0,   newcap=1,  ratio=Inf
// len=2,   oldcap=1,   newcap=2,  ratio=2.00
// len=3,   oldcap=2,   newcap=4,  ratio=2.00
// len=5,   oldcap=4,   newcap=8,  ratio=2.00
// ...
// len=256, oldcap=256, newcap=512,  ratio=2.00
// len=512, oldcap=512, newcap=848,  ratio=1.66  ← 开始平滑
// len=849, oldcap=848, newcap=1280, ratio=1.51
```

### 四、扩容的三大关键边界情况

#### 边界 1：共享底层数组的陷阱

```go
s1 := make([]int, 3, 5) // len=3, cap=5
s1[0], s1[1], s1[2] = 1, 2, 3

s2 := s1               // s2 与 s1 共享同一个底层数组
s2 = append(s2, 4)     // cap 够用(5-3=2)，不扩容，直接在原数组追加

fmt.Println(s1) // [1 2 3]   ← len 还是 3，看不到 4
fmt.Println(s2) // [1 2 3 4]

// 但 s1 的 cap 范围内确实有 4：
s1 = s1[:4]
fmt.Println(s1) // [1 2 3 4] ← 扩展 len 后能看到
```

#### 边界 2：扩容后的"分离"

```go
s1 := make([]int, 3, 3) // len=3, cap=3
s1[0], s1[1], s1[2] = 1, 2, 3

s2 := s1
s2 = append(s2, 4)     // cap 不够(3==3)，触发扩容，分配新数组

s2[0] = 999
fmt.Println(s1) // [1 2 3]    ← 不受影响
fmt.Println(s2) // [999 2 3 4] ← 两者已"分家"
```

**核心规则**：`append` 后，如果 `cap` 够用，原切片和新切片共享底层数组，修改互相可见；如果触发扩容，则新切片拥有独立的底层数组，修改互不影响。

#### 边界 3：`nil` 切片与空切片的 `append`

```go
var nilSlice []int          // nil 切片，len=0, cap=0, array=nil
emptySlice := make([]int, 0) // 空切片，len=0, cap=0, array 非 nil

nilSlice = append(nilSlice, 1)   // ✅ 完全合法，底层自动分配
emptySlice = append(emptySlice, 1) // ✅ 同样合法

// 对 nil 切片 append 是安全的，这是 Go 的惯用模式
```

### 五、最佳实践

1. **预分配容量**：如果大致知道最终元素数量，用 `make([]T, 0, estimatedCap)` 避免多次扩容和内存拷贝。

```go
// 差：频繁扩容
var ids []int64
for _, user := range users {
    ids = append(ids, user.ID) // 可能触发 O(log n) 次扩容
}

// 好：一次分配
ids := make([]int64, 0, len(users))
for _, user := range users {
    ids = append(ids, user.ID) // 零扩容
}
```

2. **函数传参注意**：函数内 `append` 可能改变 `len` 甚至分配新数组，如果需要将修改传回调用方，应返回新切片或传指针。

```go
func addItem(s *[]int, item int) {
    *s = append(*s, item) // 通过指针修改调用方的 slice header
}
// 更推荐的做法：返回新切片
func addItem(s []int, item int) []int {
    return append(s, item)
}
```

3. **切片截取时注意 cap**：`s[i:j]` 的新切片 cap 是 `cap(s)-i`，如果只想截取数据而不暴露后面的容量，使用完整切片表达式 `s[i:j:k]`。

```go
s := []int{1, 2, 3, 4, 5}
part := s[1:3]   // len=2([2,3]), cap=4([2,3,4,5])
part = append(part, 100) // 会污染 s[3]!
fmt.Println(s) // [1 2 3 100 5]  ← s 被意外修改

// 使用三索引切片限制 cap
part2 := s[1:3:3] // len=2, cap=2
part2 = append(part2, 100) // 扩容，分配新数组，s 不受影响
```

### 一句话总结

**Slice 底层是 `(array指针, len长度, cap容量)` 三元组，当 `len == cap` 时 append 触发扩容（Go 1.18+ 采用从 2x 到约 1.25x 的平滑增长策略），扩容后新旧切片"分家"，各自指向独立的底层数组。**

---

### 口头简答

Slice 的底层是一个包含指针、长度和容量三个字段的结构体，它本身不存数据，只是个"视图"。当 append 时如果长度等于容量、也就是底层数组满了，Go runtime 就会分配一块更大的新数组，把旧数据拷贝过去，然后返回指向新数组的切片。扩容策略在 Go 1.18 之后做了优化，小容量时翻倍，大容量时平滑到约 1.25 倍增长，避免浪费太多内存。
